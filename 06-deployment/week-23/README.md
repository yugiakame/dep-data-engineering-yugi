## Learner Support

### Starter Script / Template

Use this README structure checklist:

```md
# <project title>
## Problem
## Audience
## Data Source
## Pipeline
## Analysis
## Dashboard
## How To Run
## Limitations
## Next Steps
```

Repo polish checklist:

```md
- Remove unused files
- Check notebook markdown headings
- Add comments only where logic is not obvious
- Confirm links and paths are correct
```

### How To Adapt This To Your Project

- Rewrite for a stranger who has never seen the project before.
- Keep the README honest about what the project currently does.
- Prefer short explanations over dense walls of text.

### Definition of Done

- The README covers problem, data, pipeline, analysis, dashboard, and how to run.
- The repo structure is clean and understandable.
- Notebooks and scripts have enough narration to follow the logic.
- The project is presentation-ready from a documentation perspective.

### Common Mistakes

- Writing a README that assumes the reader already knows the project.
- Keeping outdated screenshots, links, or commands.
- Over-commenting obvious code while leaving confusing logic unexplained.
- Forgetting to mention assumptions and limitations.

### If You’re Stuck After 2 Hours

- Ask someone else to read only the README and tell you what they still cannot understand.
- Start by fixing headings and run instructions before polishing prose.
- Remove clutter before adding more explanation.
