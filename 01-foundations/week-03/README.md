## Learner Support

### Starter Script / Template

Use this starter structure for your repo:

```text
<your-project-repo>/
├── data/
│   ├── raw/
│   └── processed/
├── notebooks/
├── output/
├── scripts/
├── README.md
└── requirements.txt
```

Example first-pass setup flow:

```bash
mkdir -p data/raw data/processed notebooks output scripts
git init
git add .
git commit -m "Initialize DEP project structure"
```

### How To Adapt This To Your Project

- Keep the folder names aligned with the DEP structure so later weeks are easier.
- Start `README.md` and `requirements.txt` now even if they are still small.
- Make your first commit meaningful enough that a reviewer can see real setup progress.

### Definition of Done

- The repo is public and visible on GitHub.
- The starter folders exist in the root of the project.
- You have at least one meaningful commit and push.
- Your README already connects the repo to your Week 1 and Week 2 work.

### Common Mistakes

- Forgetting to push after making the local commit.
- Creating a different folder structure that will conflict with later instructions.
- Leaving the repo empty except for autogenerated files.
- Treating `requirements.txt` as optional.

### If You’re Stuck After 2 Hours

- Use the DEP Starter Kit if you have not already created a clean project repo.
- Focus on getting the folder structure and first commit done before customizing anything.
- Share the repo link and a screenshot of your local folder tree if setup keeps failing.
